## Phase 2：对齐 PRD 核心接口

> 目标：生成的代码接口与 PRD 定义完全一致。

### 2.1 PluginState 枚举

**当前：** 6 个状态（Created, Initializing, Running, Stopping, Stopped, Faulted）
**PRD：** 3 个状态（Normal=0, Faulted=1, Offline=2）

- [ ] 重写 `PluginState.cs.stpl` 为 PRD 的 3 状态模型
- [ ] 更新 `PluginBase.cs.stpl` 使用 PRD 的 `PluginStateMachine` + `AllowedTransitions` 字典
- [ ] 添加 `IAsyncDisposable` 实现（5 秒超时取消）
- [ ] 添加 `EnsureNormalState()` 守卫方法

### 2.2 IMenuModule 接口

**当前：** `ModuleId`, `ModuleName`, `Icon`, `Order`, `RequiredRole`, `GetMenuItems()`
**PRD：** `MenuKey`, `DisplayNameResourceKey`, `IconKind`, `RequiredRole`, `TargetViewName`, `SortOrder`, `ParentMenuKey`

- [ ] 重写 `IMenuModule.cs.stpl` 使用 PRD 的扁平属性模型

### 2.3 IDriver 接口

**当前：** `ReadAsync(Tag tag)`, `WriteAsync(Tag tag, object value)`
**PRD：** `ReadAsync(string address)`, `WriteAsync(string address, object value)`

- [ ] 决策：保留当前 `Tag` 参数设计（更类型安全）或对齐 PRD
- [ ] 如保留当前设计，需在 PRD 偏差文档中记录

### 2.4 ICameraService 接口

**当前：** `IsConnected`, `ConnectAsync`, `DisconnectAsync`, `GrabFrameAsync`
**PRD：** `IsOpen`, `OpenAsync`, `CloseAsync`, `CaptureAsync`

- [ ] 重写 `ICameraService.cs.stpl` 使用 PRD 的方法名
- [ ] 更新 `SimCameraService.cs.stpl` 匹配新接口

### 2.5 ConnectionParams 模型

**当前：** `Host`, `Port`, `TimeoutMs`, `Dictionary<string,string> Extra`
**PRD：** `IpAddress`, `Port`, `Rack`, `Slot`, `EndpointUrl`, `TimeoutMs`, `RetryCount`, `RetryIntervalMs`

- [ ] 重写 `ConnectionParams.cs.stpl` 添加 PRD 定义的显式属性

### 2.6 UserRole 枚举

**当前：** `None=0, Viewer=1, Operator=2, Engineer=4, Administrator=8`
**PRD：** `None=0, Operator=1, Engineer=2, Administrator=7`

- [ ] 重写 `UserRole.cs.stpl` 使用 PRD 的位值

### 2.7 IUserService 接口

- [ ] `DeleteAsync` → `DeleteUserAsync`（对齐 PRD）
- [ ] 添加 `ForceChangePasswordOnNextLogin(string username)` 方法

## 开发顺序建议

```
Phase 1 (BLOCKING)     → Phase 2 (接口对齐)     → Phase 3 (缺失模板)
生成可编译项目            对齐 PRD 核心接口          补全所有驱动/相机/Pipeline 模板
    ↓                       ↓                        ↓
Phase 4 (管线完善)      → Phase 5 (内容完善)      → Phase 6 (测试补充)
VariableResolver 补充     BCrypt、定时刷新           集成测试、编译验证

```

**每个 Phase 完成后的验证：**
1. `dotnet build src/ScaffoldX.App/ScaffoldX.App.csproj` — 0 errors
2. `dotnet test tests/ScaffoldX.Core.Tests/` — all pass
3. `dotnet test tests/ScaffoldX.App.Tests/` — all pass
4. 手动测试：运行 ScaffoldX，生成一个 Collection 项目，在 VS 2022 中打开验证编译

## PRD 偏差记录

以下为当前实现与 PRD 的有意偏差（设计改进）：

| 偏差点 | 当前实现 | PRD 定义 | 原因 |
|--------|----------|----------|------|
| IDriver 参数 | `Tag` 对象 | `string address` | 类型更安全，编译时检查 |
| Tag 模型 | 有 `Unit`, `Description` | 无 | 工业场景需要工程单位和描述 |
| ConnectionParams | `Dictionary Extra` | 显式属性 | 通用性更好，支持未知协议扩展 |
| UserRole | 添加 `Viewer` | 无 Viewer | 只读角色在工业场景中常见 |
| PluginState | 6 状态 | 3 状态 | 更细粒度的状态管理 |

**决策：** Phase 2 中需要逐一确认保留偏差还是对齐 PRD。建议保留设计改进，在 PRD 偏差文档中记录。
